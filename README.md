# OmniList Backend API

The backend for OmniList, a Mini Multi-Tenant Property Listing Platform. This serves as a secure, fast, and scalable foundation built specifically for the Intern Staff Developer Full-Stack Practical Exam.

## 🚀 Tech Stack

- **Framework:** NestJS 11 (TypeScript)
- **Database:** PostgreSQL accessed via Prisma 7 ORM
- **Authentication:** JWT (JSON Web Tokens) with Passport strategy + bcrypt for hashing
- **Validation:** Class-validator & Class-transformer
- **Files:** Local Multer upload (scalable to S3) with `ServeStaticModule` for `/uploads/` access
- **Security:** Helmet, ThrottlerModule for rate-limiting, CORS configured for frontend
- **Documentation:** OpenAPI / Swagger UI auto-generated

## ✅ Key Exam Features Implemented

1. **Role-Based Access Control (RBAC):** `ADMIN`, `OWNER`, and `USER` roles strictly enforce operations based on guards.
2. **Filtering & Pagination:** `/api/properties` implements robust limit/page pagination, price range filters, status filters, and case-insensitive keyword searches.
3. **Soft Deletes:** Deleting properties sets `deletedAt`, retaining records for admins. Safe-delete on users cascades soft deletes to owned properties.
4. **Publishing Transaction Logic:** Cannot publish unless image count > 0, price > 0, and fields are complete. Publishes transactionally.
5. **Secure Image Handling:** Local image endpoints validate MIME type (jpeg/png/webp) and size limitations (5MB limit).
6. **Robust Error Handling:** Global ValidationPipes with `whitelist` filtering.

## ⚙️ Quick Start

Install dependencies:
```bash
npm install
# Also make sure the required additions are installed:
npm install @nestjs/swagger swagger-ui-express @nestjs/throttler @nestjs/serve-static
npm install --save-dev @types/multer ts-node
```

Sync Database and Seed initial Admin user:
```bash
npx prisma db push
npm run prisma:seed
```

Start the application:
```bash
npm run start:dev
```
